## 2. 品牌规范文档解析流程

### 2.1 流程图

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   用户上传    │ ──▶ │   文档解析    │ ──▶ │   LLM提取    │ ──▶ │   规范存储    │
│  PDF/PPT文件  │     │  (文本提取)   │     │  (结构化)    │     │  (持久化)    │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
     二进制              纯文本              JSON对象           BrandRules
     bytes              string              dict               Pydantic
```

### 2.2 详细步骤

#### Step 1: 文件上传

| 项目 | 说明 |
|------|------|
| **入口** | GUI 规范管理页面上传按钮 |
| **输入格式** | PDF / PPT / PPTX 文件 |
| **代码位置** | `gui/pages/settings_page.py` |

#### Step 2: 文档解析 (文本提取)

**PDF解析** - 使用 PyMuPDF

```python
import fitz  # PyMuPDF

doc = fitz.open(stream=file_data, filetype="pdf")
for page in doc:
    text = page.get_text()
    text_content.append(f"=== 第{page_num + 1}页 ===\n{text}")

full_text = "\n\n".join(text_content)
```

**PPT/PPTX解析** - 使用 python-pptx

```python
from pptx import Presentation

prs = Presentation(io.BytesIO(file_data))
for slide in prs.slides:
    for shape in slide.shapes:
        if hasattr(shape, "text"):
            slide_texts.append(shape.text.strip())
```

#### Step 3: LLM结构化提取

| 项目 | 说明 |
|------|------|
| **API** | 豆包 LLM API |
| **SDK** | LangChain ChatOpenAI |
| **输入** | 纯文本 (最多15000字符) |
| **输出** | 结构化JSON对象 |
| **代码位置** | `src/services/document_parser.py` |

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="doubao-seed-2-0-pro-260215",
    openai_api_base="https://ark.cn-beijing.volces.com/api/v3",
    openai_api_key="<API_KEY>",
    temperature=0.1,
)

# System Prompt 定义输出结构
system_prompt = """你是品牌规范文档解析专家。请提取结构化规则，输出JSON格式：
{
  "brand_name": "品牌名称",
  "color": { "primary": {...}, "secondary": [...], "forbidden": [...] },
  "logo": { "position": "top_right", "size_range": {"min": 8, "max": 15} },
  "font": { "allowed": [...], "forbidden": [...] }
}"""

response = llm.invoke([
    SystemMessage(content=system_prompt),
    HumanMessage(content=f"请从以下品牌规范文档中提取结构化规则：\n{text[:15000]}")
])
```

**LLM输出示例：**

```json
{
  "brand_name": "示例品牌",
  "color": {
    "primary": {"name": "品牌蓝", "value": "#00A4FF"},
    "secondary": [{"name": "白色", "value": "#FFFFFF"}],
    "forbidden": [{"name": "红色", "value": "#FF0000", "reason": "与品牌调性冲突"}]
  },
  "logo": {
    "position": "top_right",
    "size_range": {"min": 5, "max": 10},
    "safe_margin_px": 20
  },
  "font": {
    "allowed": ["思源黑体", "苹方"],

  }
```

## 3. 设计图片审核流程

### 3.1 流程图

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
```

### 3.2 详细步骤

#### Step 1: 图片接收

| 项目 | 说明 |
|------|------|
| **入口** | GUI 审核页面图片拖拽/选择 |
| **支持格式** | PNG, JPEG, JPG, GIF, BMP, WEBP |
| **代码位置** | `gui/widgets/image_drop_area.py` |

#### Step 2: 图片预处理

| 项目 | 说明 |
|------|------|
| **工具** | Pillow (PIL) |
| **功能** | 格式转换、尺寸压缩、Base64编码 |
| **代码位置** | `src/services/audit_service.py:35-72` |

```python
from PIL import Image
from io import BytesIO
import base64

# 1. 读取图片
img = Image.open(BytesIO(image_data))

# 2. 格式转换（RGBA→RGB for JPEG）
if image_format in ["jpeg", "jpg"] and img.mode in ["RGBA", "P"]:
    background = Image.new("RGB", img.size, (255, 255, 255))
    background.paste(img, mask=img.split()[-1])
    img = background

# 3. 尺寸压缩（超过4K则缩放）
max_dimension = 4096
if max(img.size) > max_dimension:
    ratio = max_dimension / max(img.size)
    img = img.resize(new_size, Image.Resampling.LANCZZOS)

# 4. 转换为Base64
buffer = BytesIO()
img.save(buffer, format="PNG", quality=95)
image_base64 = base64.b64encode(buffer.getvalue()).decode()
```

|------|------|

| **消息结构** | SystemMessage + HumanMessage (文本+图片) |
| **代码位置** | `src/services/llm_service.py:182-223` |

```python

# 2. 构建System Message（审核标准）

【审核标准】
```

## 5. 大模型调用详情

### 5.1 两种LLM调用场景

| 场景 | 调用类型 | 输入 | 输出 | 模型要求 |
|------|----------|------|------|----------|
| 规范文档解析 | **纯文本LLM** | 文档文本 | 结构化JSON | 文本模型即可 |
| 图片审核 | **多模态LLM** | 文本+图片 | 审核报告JSON | **必须多模态** |

### 5.2 为什么图片审核需要多模态模型？

图片审核需要模型完成以下**视觉理解任务**：

| 任务 | 说明 | 纯文本模型能否完成 |
|------|------|-------------------|
| **颜色识别** | 识别图片中的主要颜色、色值、占比 | ❌ 无法"看"图片 |
| **Logo检测** | 判断Logo位置、大小、是否变形 | ❌ 无法识别视觉元素 |
| **OCR文字识别** | 读取图片中的文字内容 | ❌ 无法识别图像中的文字 |
| **字体判断** | 根据文字外观判断字体类型 | ❌ 需要视觉理解 |
| **布局分析** | 分析元素位置、层级关系 | ❌ 需要空间理解 |
| **风格判断** | 评估"阳光、健康、专业、生态" | ❌ 需要视觉感知 |

**结论：图片审核必须使用多模态大模型，纯文本模型无法完成视觉理解任务。**

### 5.3 规范解析可以用纯文本模型

规范文档解析的输入是**纯文本**（从PDF/PPT提取的文字），输出是结构化的品牌规则JSON。

这个任务只需要：
- 理解文本语义
- 提取关键信息
- 组织成JSON格式

因此，规范解析**可以**使用纯文本大模型（如 GPT-4、Claude、文心一言等）。
